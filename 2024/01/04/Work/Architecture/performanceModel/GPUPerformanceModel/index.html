CPU / GPU Performance Model based on Interval Analysis

!!! abstract “导言”

  在与袁福焱交流他的GPU Design Space Exploration的工作内容时，发现和我PIM模拟器Zsim, Sniper的原理是异曲同工，师出本源的方法。

简介

虽然这些模拟器不是cycle-accuracy的，但是最终误差距离cycle-accuracy竟然只有10%[^3]

基础

Interval analysis

Interval analysis is a relatively accurate performance modeling technique, which traverses the instruction trace and uses functional simulators, e.g., cache simulator, to track the stall events that cause performance loss.[^1]

high Speed and better accuracy

It shows hundred times(97x) of speedup compared to detailed timing simulations and better accuracy compared to pure analytical models.[^1]

从何时起，研究路径开始分叉了？以及之前的分析模型的发展

Analytical Model

  1. CPU
    1. (JPDC91) An analytical approach to performance/cost modeling of parallel computers.
    2. (MICRO94)Theoretical modeling of superscalar processor performance.
    3. (HPCA09) A First-Order Fine-Grained Multithreaded Throughput Model
  2. GPU
    1. (ISCA ’09.) An analytical model for a GPU architecture with memory-level and thread-level parallelism awareness.
    2. (PPoPP10) An adaptive performance modeling tool for gpu architectures.
    3. (HPCA ’11) A quantitative performance analysis model for GPU architectures.
    4. (PPoPP ’12) A performance analysis framework for identifying potential benefits in GPGPU applications.

CPU侧

  1. KARKHANIS,T.,AND SMITH的5篇 interval analysis 相关的文章
    1. (ISCA04) A first-order superscalar processor model.
    2. (ASPLOS’06) A performance counter architecture for computing accurate CPI components
    3. (ISPASS06) Characterizing the branch misprediction penalty.
    4. (ISCA07) Automated design of application specific superscalar processors: an analytical approach
    5. (TOCS09) A mechanistic performance model for superscalar out-of-order processors
  2. (HPCA10)Interval Simulation

GPU侧

  1. (ISCA04) A first-order superscalar processor model.
  2. (TOCS09) A mechanistic performance model for superscalar out-of-order processors.
  3. GPUMech[^1]
